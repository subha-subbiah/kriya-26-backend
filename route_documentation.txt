# Action Cards System - API Documentation

Base URL: `http://localhost:3000/kriyabe` (assuming standard mount point)

All routes below require Bearer Token Authentication (Team Role).

The `:kriyaID` in the routes refers to the team's unique Kriya ID.

---

## Action Card Routes

### 1. Claim Action Card
**POST** `/api/actionCards/claim/:kriyaID`
- **Description**: Randomly selects one card from the available 4 and adds it to the team's `claimedActionCards`.
- **Example Response**:
```json
{
  "success": true,
  "claimedCard": "Spyglass Focus"
}
```

### 2. Get Claimed Action Cards
**GET** `/api/actionCards/claimed/:kriyaID`
- **Description**: Returns all cards currently in the team's claimed inventory.
- **Example Response**:
```json
{
  "claimedActionCards": [
    "Black Pearl’s Resurgence",
    "Spyglass Focus"
  ]
}
```

### 3. Activate Action Card
**POST** `/api/actionCards/activate/:kriyaID`
- **Description**: Activates a card, applies its effect, and moves it to `usedActionCards`.
- **Body**:
```json
{
  "cardName": "Black Pearl’s Resurgence"
}
```
- **Example Response**:
```json
{
  "success": true,
  "message": "Gain 2 extra lives. Added to all problems.",
  "cardName": "Black Pearl’s Resurgence"
}
```

### 4. Get Used Action Cards
**GET** `/api/actionCards/used/:kriyaID`
- **Description**: Returns a list of all cards used by the team.
- **Example Response**:
```json
{
  "usedActionCards": [
    "Captain’s Hidden Map"
  ]
}
```

## Game Action Logic (Integrated Cards)

### Submission Solution
**POST** `/api/round2/submissions/submit/:kriyaID` (Verify mount in GameLogicRoutes)
- **Effect of Davy Jones’ Mercy**: If `ignoreNextFailedTestcase` is true and exactly 1 testcase fails, the submission is treated as `ACCEPTED`. Flag is consumed.
- **Effect of Spyglass Focus**: If `revealFailedTestcase` is true and the submission fails, the response includes `failedTestcaseIndex`. Flag is consumed.
